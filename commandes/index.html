BOÎTE À OUTILS

Commandes Git

La référence claire et rapide pour vos projets quotidiens.

⌘ K
30 commandes trouvées
01

Démarrage

git --version

Vérifie que Git est installé et affiche sa version.

Exemple
git --version
Rôle
Vérifier l'installation.
Quand
Avant toute configuration.
Impact
Lecture seule.
Erreur fréquente
Confondre Git et GitHub CLI.
Bonne pratique
Mettre Git à jour avec le gestionnaire du système.
git init

Initialise un dépôt dans le dossier courant.

Exemple
git init
Rôle
Créer un dépôt local.
Quand
Au démarrage d'un projet existant.
Impact
Ajoute le dossier caché .git sans modifier vos fichiers.
Erreur fréquente
Initialiser dans le mauvais sous-dossier.
Bonne pratique
Vérifier la racine avec pwd puis git status.
git clone <url>

Copie un dépôt distant et son historique.

Exemple
git clone https://github.com/user/projet.git
Rôle
Copier un dépôt et son historique.
Quand
Pour rejoindre un projet distant.
Impact
Crée le dossier, le dépôt local et le remote origin.
Erreur fréquente
Cloner dans un dossier déjà rempli.
Bonne pratique
Cloner puis lire README et branches avant de modifier.
git config

Configure l'identité et le comportement de Git.

Exemple
git config --global user.name "Ada Lovelace"
Rôle
Définir identité et comportement.
Quand
À l'installation ou par projet.
Impact
Influence l'auteur des commits et les préférences Git.
Erreur fréquente
Configurer un e-mail différent de GitHub.
Bonne pratique
Contrôler avec git config --list --show-origin.
02

Changements

git status

Affiche l'état du répertoire et du staging.

Exemple
git status --short
Rôle
Comprendre l'état courant.
Quand
Avant et après chaque action importante.
Impact
Ne modifie rien ; indique branche, staging et fichiers.
Erreur fréquente
Ignorer les messages et agir à l'aveugle.
Bonne pratique
En faire le tableau de bord du workflow.
git add

Ajoute des changements à la zone de staging.

Exemple
git add src/index.js
Rôle
Préparer le prochain commit.
Quand
Après avoir relu une modification.
Impact
Copie la version choisie dans le staging.
Erreur fréquente
Ajouter tous les fichiers sans inspection.
Bonne pratique
Préférer git add <fichier> ou git add -p.
git commit

Enregistre un instantané dans l'historique.

Exemple
git commit -m "feat: add navigation"
Rôle
Enregistrer un snapshot.
Quand
Quand un changement cohérent est prêt.
Impact
Crée un objet durable dans l'historique local.
Erreur fréquente
Mélanger plusieurs intentions.
Bonne pratique
Écrire un message précis et garder le commit atomique.
git diff

Compare les changements non validés.

Exemple
git diff --staged
Rôle
Comparer des versions.
Quand
Avant add, commit, merge ou review.
Impact
Lecture seule ; montre les lignes ajoutées et retirées.
Erreur fréquente
Oublier --staged après git add.
Bonne pratique
Lire les deux vues : git diff et git diff --staged.
git restore

Restaure un fichier ou retire du staging.

Exemple
git restore --staged index.html
Rôle
Restaurer un fichier ou retirer du staging.
Quand
Pour annuler un changement non commité.
Impact
Peut écraser le contenu local ciblé.
Erreur fréquente
Restaurer sans sauvegarder un travail utile.
Bonne pratique
Vérifier git diff avant toute restauration.
03

Historique

git log

Parcourt l'historique des commits.

Exemple
git log --oneline --graph --all
Rôle
Explorer l'historique.
Quand
Pour enquêter ou préparer une release.
Impact
Lecture seule.
Erreur fréquente
Lire uniquement les messages sans graphe.
Bonne pratique
Utiliser --oneline --graph --decorate --all.
git show

Affiche le détail d'un objet Git.

Exemple
git show HEAD
Rôle
Inspecter un commit ou tag.
Quand
Pour comprendre une modification précise.
Impact
Lecture seule.
Erreur fréquente
Confondre SHA et nom de fichier.
Bonne pratique
Commencer par git show --stat <sha>.
git blame

Indique l'origine de chaque ligne.

Exemple
git blame src/app.js
Rôle
Identifier l'origine de chaque ligne.
Quand
Pour retrouver le contexte d'une décision.
Impact
Lecture seule.
Erreur fréquente
Chercher un responsable plutôt qu'une explication.
Bonne pratique
Ouvrir ensuite le commit et la PR associés.
git reflog

Retrouve les déplacements locaux de HEAD.

Exemple
git reflog
Rôle
Retrouver les mouvements locaux de HEAD.
Quand
Après reset, rebase ou commit perdu.
Impact
Lecture seule et locale.
Erreur fréquente
Attendre trop longtemps avant la récupération.
Bonne pratique
Créer immédiatement une branche sur le SHA retrouvé.
04

Branches & intégration

git branch

Liste, crée ou supprime les branches.

Exemple
git branch -d feature/login
Rôle
Lister ou gérer les branches.
Quand
Pour inspecter, créer ou supprimer.
Impact
Déplace ou crée des pointeurs, pas les commits.
Erreur fréquente
Supprimer une branche non fusionnée.
Bonne pratique
Vérifier avec --merged avant -d.
git switch

Change de branche ou en crée une.

Exemple
git switch -c feature/login
Rôle
Changer de branche.
Quand
Pour isoler une tâche.
Impact
Déplace HEAD et adapte les fichiers.
Erreur fréquente
Changer avec des modifications incompatibles.
Bonne pratique
Créer avec git switch -c feature/nom.
git checkout

Ancienne commande pour changer de branche ou restaurer un fichier.

Exemple
git checkout main
Rôle
Ancienne commande polyvalente.
Quand
Pour maintenir des scripts historiques.
Impact
Peut changer de branche ou restaurer selon les arguments.
Erreur fréquente
Confondre les deux usages.
Bonne pratique
Préférer switch et restore pour être explicite.
git merge

Intègre une branche dans la branche courante.

Exemple
git merge feature/login
Rôle
Intégrer une branche.
Quand
Quand une fonctionnalité validée rejoint sa cible.
Impact
Combine les historiques et peut créer un commit.
Erreur fréquente
Fusionner depuis la mauvaise branche.
Bonne pratique
Lire la branche courante avant le merge.
git rebase

Rejoue des commits sur une nouvelle base.

Exemple
git rebase main
Rôle
Rejouer des commits sur une nouvelle base.
Quand
Pour actualiser une branche locale.
Impact
Réécrit les SHA des commits rejoués.
Erreur fréquente
Rebaser une branche partagée sans coordination.
Bonne pratique
Réserver le rebase aux commits locaux maîtrisés.
05

Dépôts distants

git remote

Gère les connexions aux dépôts distants.

Exemple
git remote add origin <url>
Rôle
Gérer les dépôts distants.
Quand
Pour connecter ou inspecter GitHub.
Impact
Modifie la configuration locale, pas le serveur.
Erreur fréquente
Écraser une URL sans la vérifier.
Bonne pratique
Utiliser git remote -v avant set-url.
git fetch

Télécharge les références sans les intégrer.

Exemple
git fetch --prune origin
Rôle
Télécharger les références distantes.
Quand
Avant de comparer ou intégrer.
Impact
Met à jour origin/* sans toucher à la branche courante.
Erreur fréquente
Croire que les fichiers locaux sont fusionnés.
Bonne pratique
Fetch d'abord, inspecter ensuite.
git pull

Télécharge puis intègre les changements.

Exemple
git pull --rebase origin main
Rôle
Télécharger puis intégrer.
Quand
Quand on accepte l'intégration immédiate.
Impact
Équivaut à fetch puis merge/rebase.
Erreur fréquente
Pull avec un répertoire sale.
Bonne pratique
Préférer pull --rebase sur une branche personnelle propre.
git push

Publie les commits vers un dépôt distant.

Exemple
git push -u origin main
Rôle
Publier les commits.
Quand
Après tests et commits locaux.
Impact
Met à jour une branche distante si les règles l'autorisent.
Erreur fréquente
Forcer sur une branche partagée.
Bonne pratique
Configurer l'upstream avec -u au premier push.
06

Outils avancés

git stash

Met de côté des changements temporaires.

Exemple
git stash push -m "WIP"
Rôle
Mettre temporairement de côté.
Quand
Pour interrompre un travail court.
Impact
Stocke des changements hors de la branche.
Erreur fréquente
Utiliser le stash comme stockage durable.
Bonne pratique
Nommer avec push -m et inclure -u si nécessaire.
git tag

Marque un commit comme une version.

Exemple
git tag -a v1.0.0 -m "Release"
Rôle
Marquer une version.
Quand
Après validation d'une release.
Impact
Crée une référence stable sur un commit.
Erreur fréquente
Déplacer un tag déjà publié.
Bonne pratique
Utiliser un tag annoté et SemVer.
git revert

Crée un commit qui annule un commit existant.

Exemple
git revert a1b2c3d
Rôle
Annuler par un nouveau commit.
Quand
Pour un changement déjà partagé.
Impact
Préserve l'historique et ajoute l'inverse.
Erreur fréquente
S'attendre à effacer le commit original.
Bonne pratique
Préférer revert sur main.
git reset

Déplace HEAD ou modifie le staging.

Exemple
git reset --soft HEAD~1
Rôle
Déplacer HEAD ou modifier le staging.
Quand
Pour corriger un historique local.
Impact
Selon le mode, conserve ou détruit les changements.
Erreur fréquente
Utiliser --hard sans sauvegarde.
Bonne pratique
Commencer par --soft et vérifier le reflog.
git cherry-pick

Applique un commit précis sur la branche courante.

Exemple
git cherry-pick a1b2c3d
Rôle
Appliquer un commit précis.
Quand
Pour transférer un correctif ciblé.
Impact
Crée un nouveau commit avec un nouveau SHA.
Erreur fréquente
Copier une longue série de commits liés.
Bonne pratique
Préférer merge/rebase quand toute la branche est utile.
git clean

Supprime les fichiers non suivis après prévisualisation.

Exemple
git clean -nd
Rôle
Supprimer les fichiers non suivis.
Quand
Pour nettoyer des fichiers générés.
Impact
Suppression irréversible par Git.
Erreur fréquente
Lancer -fd sans aperçu.
Bonne pratique
Toujours commencer par git clean -nd.
git rm

Supprime un fichier suivi et prépare cette suppression.

Exemple
git rm ancien-fichier.js
Rôle
Supprimer et préparer la suppression.
Quand
Quand un fichier suivi doit disparaître.
Impact
Supprime du disque et du staging, sauf --cached.
Erreur fréquente
Utiliser rm seul puis oublier le staging.
Bonne pratique
Employer --cached pour conserver localement.
git mv

Déplace ou renomme un fichier suivi.

Exemple
git mv app.js src/app.js
Rôle
Déplacer ou renommer un fichier suivi.
Quand
Pour une réorganisation.
Impact
Prépare déplacement et suppression en une action.
Erreur fréquente
Croire que Git stocke explicitement le renommage.
Bonne pratique
Vérifier le diff et les imports après déplacement.
