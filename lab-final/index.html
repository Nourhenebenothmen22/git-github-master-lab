Votre mission : créer un mini portfolio et piloter tout son cycle de développement avec un workflow Git & GitHub professionnel.

13 étapes~2h estiméesAvancé niveau
MISSION

Portfolio v1.0.0

Un projet web publié sur GitHub, développé avec plusieurs branches et un historique propre.

HTMLCSSJavaScriptGit
BRIEF

Votre cahier des charges

Le portfolio doit présenter votre profil, vos compétences, trois projets et un moyen de contact. Le design doit être responsive et inclure une petite interaction JavaScript.

Aucune correction fournie

Ce lab valide votre autonomie. Appuyez-vous sur les cours et la Cheat Sheet, comme dans un vrai projet.

Critères de réussite

  • Historique lisible et commits atomiques
  • Branches correctement fusionnées
  • README clair et projet documenté
  • Dépôt GitHub public et tag v1.0.0
  • Conflit résolu sans perte de contenu
FEUILLE DE ROUTE

Les 13 étapes

01

Créer le projet local

Créez un dossier portfolio-git et placez-vous dedans.

02

Initialiser Git

Transformez le dossier en dépôt Git et vérifiez son état.

03

Créer les fichiers

Ajoutez index.html, style.css, script.js et README.md avec un contenu initial.

04

Premier commit

Préparez tous les fichiers et créez un commit initial clair.

05

Créer deux branches

Créez feature-design et feature-script à partir de main.

06

Commits par branche

Ajoutez le style sur feature-design et l'interactivité sur feature-script.

07

Fusionner les branches

Revenez sur main et intégrez les deux fonctionnalités.

08

Connecter GitHub

Créez un dépôt vide sur GitHub et ajoutez-le comme remote origin.

09

Pousser le projet

Publiez main et configurez son upstream.

10

Créer une Pull Request

Ajoutez une amélioration via une nouvelle branche et proposez-la en PR.

11

Ajouter .gitignore

Ignorez les dépendances, fichiers système et variables d'environnement.

12

Créer le tag v1.0.0

Marquez la première version stable avec un tag annoté et publiez-le.

13

Résoudre un conflit

Créez deux modifications incompatibles, fusionnez puis résolvez proprement le conflit.

OBJECTIF FINAL

Un dépôt dont vous êtes fier.

Quand les 13 étapes sont terminées, relisez l'historique et partagez l'URL de votre projet.
